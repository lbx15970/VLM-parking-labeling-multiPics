# 多模型全量对比测试报告 (20260307 12:06:38)

## 1. 概述与结论总览

**核心结论：**
* 综合表现最好的模型是 **qwen-vl-max**，在精度 (67.3%)、召回率 (53.4%) 以及 IoU (0.748) 上全面领先，且**耗时显著最短** (23.3s)。
* **seed2.0-pro** 作为次优选择，达到了 51.7% 精度和 43.3% 召回率，各项指标均领先于上一代 seed1.8 以及开源版 qwen3-vl。
* **seed1.8** 在加入 `parking_spots` 支持后，召回率为 33.6%，但误检 (FP) 最高。
* **qwen3-vl** 在处理此全量测试（带有大量车位和标注）时表现不佳，精度仅 26.1%，且消耗了最多的 Token (约 1.4W/次)。

**测试背景:**
- 目的：测试和对比四大视觉大模型 (`qwen-vl-max`, `qwen3-vl`, `seed1.8`, `seed2.0-pro`) 在完整的 20 张高清地下车库监控图片上的识别效果。
- 关键变更：将 Ground Truth 数据迁移到 JSON 格式，并**新增了 `parking_spots` 作为识别目标参与评估**，同步更新了可视化程序的框线宽度以支持大图清晰判定。

## 3. 分析与发现

1. **qwen-vl-max 的绝对碾压优势**：
   在 20 张极其复杂且分辨率较高的地库图像中，`qwen-vl-max` 展示了首屈一指的统治力：不仅框的准确度更高（IoU 0.748），能够同时找出最多的有效目标，且推理**速度是其它模型的 3-4 倍**（仅需 23.3 秒）。不过它有 3 次遇到了 API 请求超时，可能由于瞬时的高并发调用或者底层链路限制所致。
   
2. **seed 家族的稳步提升**：
   `seed2.0-pro` 较于前代 `seed1.8` 在精度上实现了跨越（从 33% 提升至 51%），有效降低了“无中生有”的误检框（FP 从 11.8 降低到了 7.1），反映了模型对于视觉边界和文字要素的判断更加严谨准确。同时 Seed2.0-pro 也是本次测试中**最节省 Token** 的模型（单次 6992 个），侧面说明其输出十分精简克制。

3. **qwen3-vl 开源版的表现乏力**：
   与 qwen-vl-max 的亮眼表现形成鲜明对比的是，qwen3-vl (开源) 似乎对大量复杂标注指令（尤其是加上大规模 `parking_spots` 车位寻找能力后）适应不良。漏检 (FN: 14.3) 为全部模型中最高，意味着它经常直接跳过或无法发现大多数目标。并且由于输出冗杂，单次消耗近 1.4 万 token，效率垫底。

4. **历史 Bug 的修复验证 (可视化与评测逻辑)**：
   本次测试前，由于此前的 Ground Truth `parse` 方法缺失了对新加入的 `parking_spots` 标签的匹配，导致所有的正确预测依然会被视作误判，拉低了整体分数。本次测试中成功修复了该评测盲点，并将可视化图的预测框宽增加到 4px、GT绿框增加到 3px，使得最终的可视化图片输出具有高度的对齐辨识度。

## 4. 下一步优化建议
1. 生产方案选择：如果在生产环境下使用，首推闭源商业大杯 **qwen-vl-max**；如果追求极致成本并容忍稍长的推断时间，**seed2.0-pro** 同样是非常优秀的高性价比备选。
2. 提高请求稳定性：针对 `qwen-vl-max` 偶发的 Timeout（超时）现象，后续可适度在请求层加入 retry 机制，或者调低一点 `-max-concurrent` 降低瞬时高压防止失败。
