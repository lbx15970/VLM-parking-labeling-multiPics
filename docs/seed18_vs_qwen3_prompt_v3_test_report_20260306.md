# Seed-1.8 vs Qwen3-VL-Plus —— Prompt 优化 v3 测试对比报告

## 概述

- **对比对象**: Doubao Seed-1.8 vs Qwen3-VL-Plus
- **测试输入**: 2 张图片（img1.jpg、img2.jpg），每张图片每组运行 3 次，5 线程并发
- **Prompt 对比**:
  - 基线：`prompt优化.md`（已有版本，车位号码被排除在检测范围外）
  - 新版：`prompt优化v3.md`（在原基础上新增车位号码 `parking_spots` 字段检测）
- **测试时间**: 2026-03-06
- **数据来源**:
  - `outputs/csv/qwen3-vl_prompt优化_20260306_184922.csv`（Qwen3 基线）

  - `outputs/csv/qwen3-vl_prompt优化v3_20260306_195039.csv`（Qwen3 v3）
  - `outputs/csv/seed1.8_prompt优化v3_20260306_195039.csv`（Seed-1.8 v3）

## Prompt 优化 v3 变更说明

在 `prompt优化.md` 基础上做了如下**最小化修改**，未改动其余模块：

1. **新增 `parking_spots` 字段**（第2节），要求模型独立检测地面/墙面/立柱上的车位号码：
   - 提取范围：明确表示停车位编号的数字或字母数字组合（如 "023"、"B-12"）
   - 强制检出所有清晰可见的车位编号，精准框选编号本身
   - 文字清晰可见才框选；仅有图形标识但无文字，不框选

2. **`words` 字段排他修改**：车位号码若已归入 `parking_spots`，不再在 `words` 字段中重复提取

3. **核心规则第1句**：原有"严格排除地面停车位内数字"改为不再排除车位号码，仅排除车牌、车身图文及停车线

4. **最终校验规则**新增：复查车位号码是否漏检

### 综合结论

| 维度 | Seed-1.8 | Qwen3-VL-Plus |
|------|:--------:|:-------------:|
| Prompt v3 指令遵循 | ✅ 良好，新字段检测有效 | ❌ 较差，img2 几乎失效 |
| 整体精度 | **优** | 差 |
| 整体召回率 | **优** | 差 |

**总体建议**：在含车位号码的停车场场景标注任务中，`prompt优化v3` 配合 **Doubao Seed-1.8** 表现最优，两项指标（精度、召回率 img2 均达 75% 左右），且 Token 消耗更低。Qwen3-VL-Plus 对 prompt 中新增多字段结构的遵循能力不足，不建议在当前 prompt v3 下使用。

## IoU 指标口径

- 标注来源：`data/annotations/*.xml`
- bbox 后处理：模型输出坐标转换为像素坐标，裁剪至图像范围
- IoU 计算：对每个预测框，取与所有 GT 框中 IoU 最大者；使用 IoU≥0.5 阈值判定 TP
- 精度 = TP / 预测总数；召回率 = TP / GT 总数；平均 IoU 只统计 TP 匹配框
