# Seed-1.8 vs Qwen3-VL-Plus —— Prompt 优化 v3 测试对比报告

## 概述

- **对比对象**: Doubao Seed-1.8 vs Qwen3-VL-Plus

- **Prompt**: `prompt优化v3.md`（在 `prompt优化.md` 基础上新增车位号码 `parking_spots` 字段检测）
- **测试时间**: 2026-03-06 20:02
- **本次执行**: 共 20 次计划运行，成功 20 次，无失败

## 结论摘要

- **Seed-1.8 全面优于 Qwen3-VL-Plus**，在精度、召回率、IoU 三项核心指标上均领先
- img2.jpg 上差距尤为显著，Seed-1.8 精度 70.5% vs Qwen3 37.5%，召回率 68.8% vs 28.7%
- Seed-1.8 Token 消耗更低（约低 33~40%），推理耗时相当
- Prompt v3 对 Seed-1.8 全面正向；对 Qwen3 img1 影响有限，img2 上精度/召回较基线 prompt 持平或微增（此前 3 次测试显示波动更大）

---

## Prompt 优化 v3 变更说明

在 `prompt优化.md` 基础上做了如下**最小化修改**：

1. **新增 `parking_spots` 字段**：要求模型独立检测地面/墙面/立柱上的车位号码
2. **`words` 字段排他**：车位号码归入 `parking_spots`，不再在 `words` 中重复提取
3. **核心规则调整**：不再排除"地面停车位内数字"，仅排除车牌、车身图文及停车线
4. **清晰度约束**：文字清晰可见才框选；仅有图形标识但无文字，不框选

## IoU 指标口径（与脚本一致）

- 标注来源：`data/annotations/*.xml`
- bbox 后处理：模型输出坐标转到像素坐标，裁剪至图像范围
- IoU 计算：对每个预测框，取与所有 GT 框中 IoU 最大者；使用 IoU ≥ 0.5 阈值判定 TP
- 精度 = TP / 预测总数；召回率 = TP / GT 总数；平均 IoU 只统计 TP 匹配框
